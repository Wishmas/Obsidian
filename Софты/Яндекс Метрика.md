### Этап 1 — Сбор данных 
- **Начало сбора данных**
    - Данные фиксируются при переходе посетителя на сайт (из поисковой системы, рекламы, ссылок и т. д.).
    - В браузер загружается HTML-код страницы, содержащий код счётчика Метрики.
- **Первичная передача данных**
    - Первым отправляется уведомление о загрузке страницы.
    - Включает информацию:
        - Адрес предыдущей страницы
        - Браузер и операционную систему пользователя
        - Разрешение экрана и другие параметры
- **Дополнительный сбор информации**
    - Код Метрики отслеживает активность:
        - Если пользователь остаётся на странице более 15 секунд — фиксируется продолжение просмотра.
        - Регистрируются важные действия (оформление заказа, клики, прокрутка, движения мыши).
- **Идентификация пользователей**
    - Каждому посетителю присваивается уникальный идентификатор (по браузеру).
    - Позволяет формировать цепочку событий в рамках одного визита.
- **Хранение данных**
    - Вся информация о визите сохраняется в базе данных ClickHouse.
    - Основная единица учёта — визит и все события, связанные с ним.
### Этап 2 — Построение отчётов 
- **Извлечение данных**
    - Данные берутся из базы ClickHouse.
    - Для отчёта отбираются визиты, произошедшие в заданный временной интервал.
    - Возможна фильтрация визитов (например, только переходы из соцсетей).
- **Группировка данных**
    - Визиты разделяются на группы по выбранному параметру (например, источник трафика).
    - Для каждой группы рассчитываются метрики:
        - Общее число визитов
        - Количество уникальных посетителей
        - Среднее время просмотра и другие показатели
- **Формирование отчёта**
    - Итоговый отчёт представлен в виде таблицы, где каждая строка — это группа визитов.
    - Данные можно визуализировать с помощью диаграмм.
    - Доступен анализ динамики изменения метрик во времени.

## Цели

**Цель** — это действие посетителя, в котором заинтересован владелец сайта: просмотр определенного количества страниц, посещение конкретной страницы, нажатие кнопки, переход по ссылке, оплата заказа и т. д.

Когда пользователи приходят на ваш сайт и взаимодействуют с ним, Яндекс Метрика собирает информацию об этом и фиксирует достижение целей. На основе собранных данных рассчитываются т. н. целевые метрики:
- **Достижение цели** — это момент, в который посетитель выполнил заданное в параметрах цели условие.
- **Целевой посетитель** — это посетитель, который выполнил условие, заданное в параметрах цели.
- **Целевой визит** — это визит, в течение которого произошло достижение цели. В течение одного целевого визита может произойти несколько достижений одной и той же цели.
- **Конверсия** — это отношение количества целевых визитов к общему числу визитов.

**Конверсионная цель** позволяет просмотреть статистические данные (конверсия, достижения цели и пр.).

**Ретаргетинговая** цель предназначена для настройки показов по условиям подбора аудитории через Яндекс.Директ.
**Ретаргетинг** - то технология, позволяющая показывать объявления аудитории, которая посещала Ваш сайт и совершила на нем какие-либо действия. В нашем случае — Директ берет информацию о целях и пользователях из Яндекс.Метрики.

**Автоматические цели:**
Метрика собирает данные о различных действиях на сайте. Она может создавать цели автоматически на основе некоторых действий. Цели начинают отображаться, когда Метрика фиксирует на сайте хотя бы одно из подходящих событий.
По умолчанию создание автоматических целей включено в настройках:
- новых счетчиков;
- счетчиков, на которых еще не настроены цели;
- счетчиков, на которых цели используются редко.
